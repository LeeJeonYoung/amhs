# 실물 브링업 시행착오 기록 (라인트레이서 격자 AMHS)

> 2026-06-22~23, robot_grid 펌웨어를 실제 라인트레이서 3대 + nRF24 허브 위에서 돌리며 겪은 문제와 해결.
> 시뮬(알고리즘)에서 검증한 것을 **실물 제어 수준**으로 내리며 마주친 현실 변수들. 면접 방어 카드.

핵심 한 줄: **"격자 주행이 안 됐던 진짜 원인은 알고리즘이 아니라, 센서 극성·회전 기구학·전원 안정성 같은 물리 계층이었다."**

---

## 1. IR 라인센서 극성 (LINE_ON) — 로봇마다 다름
- 증상: 흰 바닥인데 robot_grid가 출발 즉시 멈춤(교차로 오인) 또는 라인 인식 못 함.
- 측정(02_ir_test, analog raw): **흰 바닥 raw≈25(digital 0), 검정 테이프 raw≈1023(digital 1)** → 이 센서는 **검정=HIGH**.
- robot_grid 기본 `LINE_ON=LOW`(검정=LOW)와 **반대** → `-DLINE_ON=HIGH`로 빌드해야 함.
- 교훈: **극성은 센서/배선마다 다르다.** 펌웨어에 `#ifndef LINE_ON` 두고 빌드플래그로 주입. 로봇별 02_ir_test로 실측 필수.
- 함정: **센서가 너무 가까우면(부품 빼서 바닥에 붙음) 반사가 포화**돼 흰/검정 구분 0 (raw 둘 다 동일). 적정 높이 5~15mm.

## 2. ★조향 반전 버그 — 라인추종이 "전혀" 안 됐던 진짜 원인
- 증상: RUN 주면 라인을 못 따라가고 즉시 이탈. 처음엔 IR 약함으로 의심.
- 사용자 관찰이 결정적: *"왼쪽 센서에 검정 닿으면 왼쪽으로 틀어버림 — 반대로 틀어야 하는데."*
- 원인: `lineFollow`의 좌/우 보정 모터 속도가 **뒤바뀌어** 있었음 → 라인에서 멀어지는 방향으로 조향 → 즉시 이탈.
- 해결(v6): `L&&!R`/`!L&&R` 두 보정 분기의 모터 속도 swap.
- 검증: `line_follow_test.ino`(무선·nRF24 없이 켜자마자 라인추종)로 단독 확인 → **라인추종 성공**.
- 교훈: "안 된다"의 원인을 한 곳으로 단정 말 것(IR로 의심했지만 조향이었음). 무선·전원을 배제한 **standalone 테스트**로 변수 격리.

## 3. 회전 기구학 — 제자리 피벗 → 전진-호(arc) 회전
- 증상: 교차로에서 우회전 명령 → ~20°만 돌다가 **후진하며 원래 직선으로 복귀** → 회전 실패.
- 원인: 제자리 피벗(한 바퀴 후진)이라, 후진 성분이 로봇을 원래 라인으로 끌고 감.
- 사용자 통찰: *"앞으로 이동하면서 회전시켜야 한다."*
- 해결(v7): pivot을 **전진-호**로 변경(양쪽 전진, 안쪽 바퀴만 느리게, 후진 없음). 새 라인 쪽으로 빠져나감.
- 회전 속도도 `TURN_SPEED 160→230`(약한 배터리에서 제자리 토크 부족 → 부스트). **회전은 cruise 속도와 독립**(고정 TURN_SPEED/TURN_MS) — cruise 낮춰도 회전 유지.

## 4. 자력출발 스톨 → 킥스타트
- 증상: 정지 상태서 RUN 줘도 **"삐이이~" 소리만 나고 안 움직임**(모터 스톨). 장애물 아님 — 정지마찰을 약한 배터리가 못 이김.
- 해결(v8): **킥스타트** — RUNNING 진입 후 첫 `KICK_MS` 동안 `KICK_SPEED(255)`로 부스트 후 cruise로. 손으로 안 밀어도 출발.
- 추가(v10): 약한 배터리에선 짧은 킥(300ms) 후 cruise에서 또 락 → `KICK_MS`를 길게(900ms). 그래도 **배터리가 죽으면 한계**(소프트로는 못 메움).

## 5. 교차로 재검출 디바운스(RENODE) — 회전 직후 가까운 교차로 누락
- 증상: 우회전 직후 바로 만나는 교차로를 못 잡음.
- 원인: `RENODE_MS=600`(통과 직후 무시 구간)이 너무 길어, 회전 후 가까운 다음 교차로가 무시 구간에 걸림.
- 해결: `RENODE_MS 600→300`. (단축하면 회전 직후 교차로도 잡힘. 너무 짧으면 같은 노드 중복 카운트 위험 — 트레이드오프.)

## 6. 복구 로직 — 길잃음/벽 → 후진 재탐색 (graceful degradation)
- 요구: *"벽에 박았으면 후진해서 복구."*
- 구현(v9): 라인 접촉(센서/교차로) 없이 `LOST_MS(4s)` 넘으면 = 길잃음/벽 → `ST_RECOVER`로 후진, **라인 재접촉 또는 시간초과(RECOVER_MS)까지** 후진 후 추종 재개.
- 의미: 실 AMHS의 "막히면 우회/후퇴"와 같은 결, 시뮬의 데드락 회복을 실물 단일 로봇 수준으로.

## 7. 속도 트레이드오프 (실측)
- **자력출발**엔 높은 토크(킥 255, cruise≳150) 필요. 너무 낮으면(120) 주행 중에도 락.
- **교차로 감지**엔 낮은 속도 유리(140 성공, 200은 휙 지나가 INTERSECT_MS 50ms 안에 못 잡음).
- 결론: 킥스타트(출발 부스트) + cruise 140~160이 절충점. 더 나은 해법 = 킥 후 점진 감속 또는 전원 안정화.

## 8. ★성과 — 격자 회전주행 완주
- `grid_drive.py 3 "S1 R1 R1 R1" 140` → **RUN→우회전→우회전→우회전→정지(목적지)** 전체 시퀀스 완주(사각형 □).
- = **라인추종 + 교차로 감지 + 다중 회전 + 경로 완주**를 실물 1대로 검증. (이전엔 8자 직진만.)

## 9. ★최종 블로커 = 전원(배터리) + 무선(nRF24) 안정성
- 통신 불안정(NO-ACK, 한 대씩만 응답, RUN 간헐 미수신), 모터 스톨, 센서등 희미 — **전부 약한 배터리** 한 증상.
- 어댑터(AMS1117, 3.3V+커패시터)가 있어도 **배터리 입력이 약하면 레귤레이터 출력도 무너져** nRF24 brownout.
- **nRF24 과열 사고**: VCC에 9V/5V 직결 → 3.3V 소자 과전압 → 반복 과열·손상. **nRF24 VCC는 반드시 3.3V(또는 어댑터 경유), 탈착은 전원 OFF.**
- 회전 비대칭(오른쪽만 약함): 좌/우 모터 토크 차 + 약한 배터리 → `TRIM` 조정 + 새 배터리로 해결.
- 다중 로봇 동시 제어의 병목 = **전원·통신 안정성**(알고리즘 아님). 실 AMHS의 핵심 과제와 동일.
