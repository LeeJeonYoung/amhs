## 0. 시작 전 3가지 약속 (꼭)
1. **전원/USB 다 뽑고 배선한다.** 다 꽂은 뒤 마지막에 전원을 넣는다. (꽂은 채로 배선하면 합선 위험)
2. **GND(검정, 마이너스)는 모든 부품이 서로 이어져야 한다.** "기준선"이라 안 이어지면 신호가 엉킨다.
3. **점퍼선 색은 자유지만 규칙을 정하자.** 빨강=전원(+), 검정=GND(−). 나중에 디버깅이 훨씬 쉽다.

## 1. 부품이 뭐가 뭔지 (역할부터)

| 부품 | 생김새 | 역할 | 비유 |
|---|---|---|---|
| **아두이노 UNO** | 파란 보드, USB 포트 | 두뇌. 센서 읽고 모터 돌림 | 사람의 뇌 |
| **모터드라이버(L298N 등)** | 방열판 달린 모듈 / 키트 쉴드 | UNO 신호로 모터에 큰 전류 공급 | 뇌의 명령을 받아 근육에 힘 주는 척수 |
| **DC 모터 ×2 + 바퀴** | 노란 기어박스 모터 | 좌/우 바퀴 굴림 | 다리 |
| **IR 라인센서 ×2** | 검정 소자 2개 + LED | 바닥 검정선 감지 | 눈 (바닥만 봄) |
| **초음파 HC-SR04** | 눈처럼 생긴 원통 2개 | 앞 장애물 거리 측정 | 더듬이 |
| **nRF24L01 + 어댑터** | 작은 모듈 + 받침 보드 | 무선 통신(허브↔로봇) | 무전기 |
| **AA 배터리 홀더** | 건전지 4개 케이스 | 모터용 힘센 전원 | 배터리팩 |

> ⚠️ **에듀이노 라인트레이서 키트**는 모터·센서·드라이버가 **이미 어느 정도 연결돼 있거나 전용 쉴드**로 꽂게 돼 있을 수 있습니다.
> 그러면 모터/센서 배선은 **키트 설명서가 1순위**예요. 우리가 *반드시 직접* 추가하는 건 **무전기(nRF24)** 하나입니다.

## 2. 큰 그림 (로봇 1대 = 이렇게 생김)

```
                    ┌─────────────────────────┐
   [IR 센서 좌]──A0─┤                         ├─D9~D13── [nRF24 무전기]
   [IR 센서 우]──A1─┤      아두이노 UNO        │
   [초음파]──A2/A3──┤        (두뇌)            ├─D2,4,5 ── [모터드라이버]──[왼바퀴]
                    │                         ├─D6,7,8 ──     "      ──[오른바퀴]
                    └───────────┬─────────────┘
                                │ 5V/GND 전원 공유
                         [AA 배터리팩] ──(모터 힘)──> 모터드라이버
```

허브는 훨씬 단순합니다:
```
[Mac] ──USB── [허브 UNO + nRF24 하나] ))) 무선 ((( [로봇1/2/3]
```
허브엔 모터·센서·배터리 **없음**. 무전기 하나 + USB 끝.

## 3. ★가장 중요★ 무전기(nRF24) 연결 — 이건 직접 해야 함

### 3-1. 먼저 모듈을 어댑터에 꽂기 (납땜 없음)
- nRF24 모듈 밑면 **8핀(2×4)**을 어댑터(AMS1117 받침보드) 위 소켓에 **그대로 끼웁니다.**
- 방향만 주의: 한쪽으로만 들어가게 돼 있어요. 핀 라벨이 보이게.
- **왜 어댑터를 쓰나?** nRF24는 3.3V만 견디는데, 어댑터가 5V를 3.3V로 낮춰줘서 **아두이노 5V에 바로 꽂아도 안전**해집니다.

### 3-2. 어댑터 → 아두이노 (암–수 점퍼선 7가닥)
어댑터 옆에 라벨이 적힌 핀이 줄지어 있습니다. 아래처럼 7개를 잇습니다.

```
 어댑터 핀        아두이노 핀      의미
 ─────────────────────────────────────────
 VCC   ───────►  5V             전원 (3.3V 아님! 어댑터가 변환)
 GND   ───────►  GND            기준선(−)
 CE    ───────►  9              "보낼까/받을까" 스위치
 CSN   ───────►  10             "이 무전기랑 얘기" 선택
 SCK   ───────►  13   ┐
 MOSI  ───────►  11   ├ SPI(데이터 통신선, 위치 고정)
 MISO  ───────►  12   ┘
 IRQ   ───────►  (연결 안 함)
```

외우는 법: **"씨이 구(CE=9), 씨에스엔 십(CSN=10), 나머지 SCK·MOSI·MISO는 13·11·12 고정."**
IRQ 한 개만 안 쓰고 나머지 7개 다 꽂으면 됩니다.

> 🔑 핵심 주의 2개
> 1. **VCC는 반드시 5V.** 3.3V에 꽂으면 신호가 약해 통신이 안 됩니다(어댑터가 알아서 낮춤).
> 2. 로봇 3대 + 허브 1대 = **똑같은 7가닥 배선 4세트**. 한 번 익히면 끝.

## 4. 모터 쪽 (모터드라이버 ↔ UNO ↔ 모터 ↔ 배터리)

> 키트에 드라이버가 포함/쉴드면 **키트 설명서대로** 꽂고, 코드 핀과 다르면 5번 참고.
> 아래는 일반 L298N 기준 표준 배선입니다.

### 4-1. 드라이버 ↔ 아두이노 (신호선 6가닥)
```
 드라이버   아두이노    역할
 ──────────────────────────────
 ENA  ───►  5 (PWM)    왼쪽 속도(세기)
 IN1  ───►  2          왼쪽 방향 ┐ 두 개로 정/역 결정
 IN2  ───►  4          왼쪽 방향 ┘
 ENB  ───►  6 (PWM)    오른쪽 속도
 IN3  ───►  7          오른쪽 방향 ┐
 IN4  ───►  8          오른쪽 방향 ┘
```
- ENA/ENB는 **PWM 핀(5,6)** 이어야 속도 조절이 됩니다.
- 방향이 반대로 돌면? → **진단 ①모터 테스트**로 확인 후, 그 쪽 IN 두 개만 바꿔 끼우면 됩니다(IN1↔IN2 또는 IN3↔IN4).

### 4-2. 드라이버 ↔ 모터 (바퀴)
- 드라이버의 **OUT1/OUT2 = 왼쪽 모터 두 선**, **OUT3/OUT4 = 오른쪽 모터 두 선**.
- 두 선 순서는 방향만 결정 → 반대면 두 선 바꿔 끼우면 됨(또는 코드로).

### 4-3. 드라이버 ↔ 배터리 (모터 힘)
```
 AA 배터리팩(+) ───► 드라이버 +12V(VS, 모터전원 입력)
 AA 배터리팩(−) ───► 드라이버 GND  ───► 아두이노 GND  (★공통 GND★)
```
- **모터는 USB가 아니라 배터리로 돌립니다.** USB만으론 힘이 모자라요.
- 배터리(−)와 아두이노 GND를 꼭 **이어주세요(공통 GND)**. 이게 빠지면 모터가 안 돌거나 제멋대로 돕니다.

## 5. 눈·더듬이 (IR 센서 + 초음파)

### IR 라인센서 2개
```
 왼쪽 센서 신호(OUT) ───► A0      ┐ 전원선 VCC→5V, GND→GND 도 각각 연결
 오른쪽 센서 신호(OUT)───► A1      ┘
```
- 센서마다 보통 VCC/GND/OUT(또는 D0) 3핀. OUT만 A0/A1로, 전원은 5V/GND로.
- 센서 높이는 바닥에서 **5~15mm**. (도착 후 ②IR 테스트로 조정)

### 초음파 HC-SR04 (4핀)
```
 VCC  ───► 5V
 Trig ───► A2     (소리 쏴!)
 Echo ───► A3     (돌아온 시간 받기)
 GND  ───► GND
```

## 6. 키트 배선이 코드랑 다르면? (당황 금지)
이 프로젝트 핀 번호는 **"키트 기본배선 가정값"**이에요. 키트 설명서를 봤더니 모터/센서가 다른 핀을 쓰면, 둘 중 하나:
- **(쉬움) 코드를 바꾼다** → `firmware/robot/robot.ino` 맨 위 `#define` 숫자만 키트에 맞게 수정.
- (번거로움) 배선을 코드에 맞춘다.
- **단, nRF24가 쓰는 9·10·11·12·13 핀은 비워둬야** 합니다. 키트가 이 핀을 쓰면 모터/센서 쪽을 다른 핀으로 옮기고 코드도 그 핀으로 수정.

## 7. 다 꽂은 뒤 점검 체크 (전원 넣기 전)
- [ ] nRF24 VCC = **5V** 에 꽂았나 (3.3V 아님)
- [ ] nRF24 7가닥(CE9·CSN10·SCK13·MOSI11·MISO12·VCC·GND) 다 맞나
- [ ] 모터드라이버 ENA/ENB가 PWM핀(5,6)인가
- [ ] **모든 GND가 하나로 이어졌나** (UNO·드라이버·배터리·센서)
- [ ] 9~13 핀에 모터/센서가 안 겹치나
- [ ] 배터리 +/− 거꾸로 안 꽂았나
- [ ] 로봇 3대 배선이 **서로 똑같나** (1대 완성 후 사진 찍어 복제)

## 8. 흔한 실수 TOP 5
1. nRF24를 3.3V에 꽂음 → 통신 안 됨. **5V로.**
2. 공통 GND 안 이음 → 모터가 안 돌거나 랜덤 동작.
3. ENA/ENB를 PWM 아닌 핀에 → 속도조절 안 됨.
4. 모터전원을 USB로만 → 힘 부족. **배터리 연결.**
5. 3대 배선이 제각각 → 1대만 되고 나머지 디버깅 지옥. **사진 떠서 복붙.**
